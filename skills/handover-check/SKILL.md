# 최종 점검 & 인수인계 (6단계)

목표: "다른 사람이 받아서 바로 운영·개발할 수 있는가?"를 보장한다. 팀장(당신)이 팀에 전파해야 하므로 **쉽고 명확하게**.

## 진행 순서

### 1. 완결성 점검
`reference/handover-checklist-ko.md`로 1~5단계 산출물을 점검:
- 요구사항·설계(아키텍처/ERD/보안)·구현·리뷰원장·테스트리포트·모니터링 계획이 **존재하고 최신**인가.
- 서로 **일관**된가(예: ERD ↔ 실제 스키마, api-spec ↔ 구현).
- CLAUDE.md의 6단계 체크박스가 실제와 맞는가.

### 2. 공백 식별
빠지거나 낡은 산출물, 미해결 결함, "가정(_assumed)"으로 남은 값들을 목록화하고 채운다.

### 3. 인수인계 문서 작성 (`docs/06-handover/handover.md`)
신규 담당자 관점으로:
- 서비스 한 줄 요약 + 아키텍처 그림 링크.
- 로컬 실행법(docker compose), 배포법, 환경변수/비밀 관리.
- 주요 폴더·모듈 지도, 자주 하는 작업.
- 운영: 대시보드·알람·런북, 장애 시 연락/절차.
- 알려진 이슈·다음 할 일.

### 4. 접근성 확인
문서 저장소(Confluence)에 정리하고 링크가 유효한지 확인(업로드 전 사용자 확인). skill.md 상태를 완료로 갱신.

## 판정
- 체크리스트 전 항목 충족 + "처음 보는 사람도 30분 내 실행 가능" 기준 통과해야 완료.

## 하지 말 것
- 산출물 나열만 하고 맥락 생략. 낡은 정보 방치.

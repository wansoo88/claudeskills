# 코드리뷰 체크리스트 (참조)

## 정확성 (fail 가능)
- [ ] 엣지케이스(빈값·null·경계·대량) 처리
- [ ] 에러 처리·예외 전파가 적절, 조용한 실패 없음
- [ ] 동시성/경쟁조건 이슈 없음
- [ ] 반환값·상태코드가 계약(API spec)과 일치

## 설계/구조
- [ ] 레이어 경계 준수(SQL은 데이터 레이어, 규칙은 도메인)
- [ ] 단일 책임, 과도한 결합/중복 없음
- [ ] 이름이 의도를 드러냄, 매직넘버 없음

## 테스트 (fail 가능: 핵심 로직 테스트 부재)
- [ ] 핵심 로직에 단위테스트 존재
- [ ] 실패 케이스도 테스트, 커버리지 합리적

## 성능
- [ ] N+1 쿼리·불필요한 반복 없음
- [ ] 대량 데이터에 스트리밍/페이지네이션

## 가독성/유지보수
- [ ] 주변 코드 스타일과 일치
- [ ] 주석은 '왜'를 설명(자명한 '무엇'은 지양)

## 보안 냄새 (발견 시 security-review로 강하게 연계)
- [ ] 하드코딩된 비밀정보 없음
- [ ] 사용자입력 검증, SQL 파라미터 바인딩

## 판정 기준
- 정확성 결함 / 보안 냄새 / 핵심 테스트 부재 / 레이어 위반 → **fail**
- 경미한 개선만 있으면 코멘트 후 **pass**

# 보안 베이스라인 (참조) — OWASP + AWS

> 근거: OWASP Top 10, OWASP ASVS(무료 공개 표준), AWS 보안 기본. 3단계 security-review의 기준.

## OWASP Top 10 (2021) 대응 요약
| # | 위협 | 핵심 대응 |
|---|---|---|
| A01 | 접근통제 실패 | RBAC, 서버측 권한검증, 객체수준 인가(IDOR 방지) |
| A02 | 암호화 실패 | TLS 전송암호화, KMS 저장암호화, 약한 해시 금지(bcrypt/argon2) |
| A03 | 인젝션(SQLi/XSS) | 파라미터 바인딩(ORM), 입력검증, 출력 인코딩 |
| A04 | 안전하지 않은 설계 | 위협 모델링, 보안 요구 명세화 |
| A05 | 보안 설정 오류 | 기본계정/포트 차단, 최소권한, 에러 상세 노출 금지 |
| A06 | 취약 컴포넌트 | 의존성 스캔(SCA), 정기 업데이트 |
| A07 | 인증 실패 | MFA, 세션 관리, 무차별 대입 방어(레이트리밋) |
| A08 | 무결성 실패 | 서명 검증, 신뢰된 CI/CD, 역직렬화 주의 |
| A09 | 로깅/모니터링 실패 | 접근·변경 감사로그, 이상탐지, 알람 |
| A10 | SSRF | 아웃바운드 검증, 메타데이터 엔드포인트 차단 |

## AWS 보안 기본
- **네트워크**: DB는 프라이빗 서브넷. 보안그룹 최소개방(ALB→App→DB). 퍼블릭 S3 금지.
- **자격/권한**: IAM 최소권한, 루트키 미사용, 역할 기반. 키 로테이션.
- **비밀정보**: Secrets Manager / Parameter Store. 코드·이미지·로그에 비밀 금지.
- **암호화**: RDS/S3/EBS KMS 저장암호화, 전 구간 TLS.
- **경계**: 외부 진입에 WAF(필요 시), Shield(대규모).
- **관측**: CloudTrail(감사), GuardDuty(위협탐지), CloudWatch 알람.

## 데이터 민감도별 (brief.sensitive_data)
- **개인정보(pii)**: 수집 최소화, 마스킹/암호화, 접근권한 분리, 보관기간·파기 정책. 개인정보보호법 매핑.
- **결제/금융(payment)**: 카드정보 직접저장 지양(PG 위임/토큰화), 강한 감사, 규제 준수.
- **사내 기밀(confidential)**: 역할기반 접근, DLP 관점, 다운로드/공유 통제.

## security-review 체크리스트 씨앗 (3단계에서 사용)
- [ ] 모든 API에 인증·인가 서버측 검증
- [ ] SQL은 파라미터 바인딩, 사용자입력 검증
- [ ] 비밀정보 하드코딩 없음(스캔 통과)
- [ ] 전송 TLS·저장 암호화 적용
- [ ] 민감정보 로그 미출력, 감사로그 존재
- [ ] 의존성 취약점 스캔 통과
- [ ] 에러 메시지에 내부정보 노출 없음

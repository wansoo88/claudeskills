# ERD / 데이터 모델링 가이드 (참조)

## 정규화 기본
- **1NF**: 원자값(반복 그룹 제거). **2NF**: 부분 함수 종속 제거. **3NF**: 이행 종속 제거.
- 기본은 3NF. 조회 성능이 병목이고 근거가 있을 때만 **의도적 비정규화**(집계 컬럼·중복 저장)하고 주석으로 이유를 남긴다.

## 키·제약
- 모든 테이블에 **PK**(가급적 surrogate `id bigint`). 자연키는 유니크 제약으로 별도 관리.
- **FK**는 '다' 쪽에 `{부모}_id`. 참조 무결성·삭제 정책(CASCADE/RESTRICT) 명시.
- 유니크·NOT NULL·CHECK로 도메인 규칙을 DB에서 강제.

## 인덱스 전략
- **조회 패턴 우선**: WHERE/JOIN/ORDER BY에 쓰는 컬럼 위주.
- FK 컬럼엔 대개 인덱스. 카디널리티 낮은 컬럼 단독 인덱스는 지양.
- 복합 인덱스는 선택도 높은 컬럼을 앞에. 커버링 인덱스로 조회 최적화.
- 남발 금지 — 쓰기 비용·용량 증가.

## 민감정보·감사 (brief.sensitive_data)
- 개인정보/결제: 컬럼 암호화(또는 토큰화), 접근권한 분리(뷰/행수준 보안).
- **감사 이력**: created_at/updated_at/created_by, 필요 시 이력 테이블(temporal).
- 소프트삭제(deleted_at) 여부를 초기에 결정.

## 시간·유형 규약
- 시간은 UTC 저장(timestamptz), 표시 시 변환. 통화·수량은 numeric(정밀도 명시).
- 상태값은 열거(enum/코드테이블)로 관리.

## Mermaid erDiagram 카디널리티
| 관계 | 표기 |
|---|---|
| 1:1 | `A ||--|| B` |
| 1:N | `A ||--o{ B` |
| N:1 | `A }o--|| B` |
| N:M | `A }o--o{ B` (조인테이블 권장) |

## 성능 점검 체크
- 대량 조회에 풀스캔 없는지(실행계획).
- N+1 조회 유발 구조 아닌지.
- 파티셔닝(시간/범위) 필요한 대용량 테이블 식별.
